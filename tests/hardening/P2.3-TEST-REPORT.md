## Covered Risks

| Risiko | Abdeckung |
|--------|-----------|
| Doppelte Migration bei Neustarts | Idempotenz: Datei wird umbenannt, zweiter Aufruf migriert nicht |
| Doppelverarbeitung bereits verarbeiteter Mentions | `recordMentionSkipped` → `isProcessed` persistiert über Cache-Reset |
| Doppelposts bei vorhandenen Published-Markern | `publishWithRetry` prüft `isPublished`, ruft Publish-Fn nicht erneut |
| Restart zwischen Publish-Schritten führt zu Doppelpost | `resetEventStates` simuliert Cache-Clear; persistierter State verhindert Republish |
| Health/Ready zeigt falsch positiv | Fehlender Heartbeat → degraded, stale → unhealthy |
| RateLimiter memory vs store inkonsistent | Beide Modi testen Capacity-Limit |
| runtimeConfig ungetestet | Smoke: Kern-Keys und Typen |

## Gaps Still Remaining

- **Cross-Process Heartbeat:** FileSystemStore schreibt `worker:last_poll_success` in `kvCache` (in-memory). Echter prozessübergreifender Test erfordert Redis.
- **Migration Edge Cases:** Defekte/leere `processed_mentions.json` (z. B. ungültiges JSON) nicht explizit getestet.
- **RateLimiter Backend-Wechsel:** `cachedStoreBackend` wird bei Wechsel von `memory` zu `store` nicht zurückgesetzt; Tests nutzen isolierte IDs, um Konflikte zu vermeiden.
- **validateEnv-Fehler:** runtimeConfig wirft bei fehlenden Env-Variablen; kein expliziter Test für Fail-Fast-Verhalten.
