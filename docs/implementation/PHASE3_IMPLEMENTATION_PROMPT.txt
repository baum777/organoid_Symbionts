# PHASE‑3 IMPLEMENTATION PROMPT

## Adaptive Character Evolution & Ensemble Dynamics

---

# SYSTEM ROLE

You are a senior software engineer operating in autonomous implementation mode inside an existing production‑oriented repository.

You are continuing the **GNOMES migration**.

PHASE‑1 introduced:

* gnome registry
* routing scaffolding
* prompt composition layer
* character memory primitives

PHASE‑2 integrated these components into the live runtime pipeline so that:

* a gnome is selected per interaction
* prompts are composed dynamically
* routing and interaction events are stored

PHASE‑3 introduces **controlled adaptive character evolution and ensemble behavior**.

The goal is to allow gnomes to develop personality nuances, relationships, and running jokes over time **without allowing uncontrolled LLM drift or identity corruption**.

# PHASE‑3 GOAL

Enable **long‑term character development** while keeping the core persona stable.

After PHASE‑3 the system must support:

* evolving gnome traits within bounded limits
* persistent running jokes and motifs
* user‑gnome relationship arcs
* character‑specific memory reinforcement
* cameo / swarm replies
* adaptive stylistic tendencies

Core identities must remain stable.

Evolution only occurs in **allowed adaptive layers**.

---

# CORE PRINCIPLE

Character identity consists of two layers:

## Stable Core

Never changes automatically:

* gnome name
* archetype
* core role
* safety boundaries
* baseline personality traits
* allowed behavior domains

These are defined in gnome profile files.

## Adaptive Surface

May evolve gradually:

* preferred phrasing
* running jokes
* relationship tone
* topic affinity
* stylistic tendencies

All changes must remain inside configured trait bounds.

# 4. CHARACTER‑SPECIFIC MEMORY RANKING

Enhance:

```
src/memory/characterMemory.ts
```

Memory ranking must consider:

* relevance to current topic
* recency
* gnome personality affinity
* relationship context

Ranking score example:

```
score = (relevance * 0.5) + (recency * 0.3) + (persona_affinity * 0.2)
```

Only the top memory fragments should be injected into prompts.
